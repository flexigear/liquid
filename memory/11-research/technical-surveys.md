# Technical Surveys

## Purpose

Store solver and implementation-option surveys for concrete technical blockers, so the project can compare approaches systematically instead of restarting research from zero each time.

## Usage Rule

- When a new technical blocker appears, create a dedicated survey file under `memory/11-research`.
- Summarize the blocker, candidate routes, ranking, sources, and the recommended proof-of-concept order.
- Update this index with a short pointer to the new survey.
- Update `memory/90-active/current-context.md` if the survey changes the active implementation plan.
